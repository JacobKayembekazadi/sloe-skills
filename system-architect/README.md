# system-architect

**Generate comprehensive system architecture from any product idea.**

Version: 2.0 | Updated: 2026-03-03

---

## What this skill does

Takes any product description, PRD, or app idea and maps it across **14 architecture layers** — from state management to security to emotional experience. Forces every blind spot to the surface before a line of code is written.

**Output:** A complete architecture document covering all 14 layers, ready to hand to an engineering team or use as a build spec.

## When to use it

- You have an app idea and need to think it through before building
- A client gives you a PRD and you need to find the gaps
- You want to see how features connect before scoping the work
- You're doing a technical discovery session with a client

## The 14 layers

| # | Layer | Question it answers |
|---|-------|-------------------|
| 1 | System Map | What state connects features? |
| 2 | Journey Map | What order must things happen? |
| 3 | Failure Map | What breaks and what catches it? |
| 4 | Rules Engine | What logic governs behavior? |
| 5 | Dependency Map | What external services are needed? |
| 6 | Validation Schema | What gates data quality? |
| 7 | Cost Model | Where does money/compute go? |
| 8 | Persistence Layer | How does data survive and sync? |
| 9 | Security Layer | Who can access what? |
| 10 | Resilience Layer | What works without network? |
| 11 | Observability Layer | How do we know it's working? |
| 12 | Accessibility Layer | Can everyone use it? |
| 13 | Evolution Layer | How do we ship changes? |
| 14 | Experience Layer | How does it feel? |

## Usage

> *"Use the system-architect skill to design [product idea]"*

> *"Run a system architecture analysis on this PRD"*
